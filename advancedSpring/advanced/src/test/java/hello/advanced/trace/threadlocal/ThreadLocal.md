# 1. 쓰레드 로컬이란?
```text
Thread Local은 해당 쓰레드만 접근 할 수 있는 특별한 저장소를 말한다. 쉽게 이야기해서 물건 보관함을 떠올리면 된다.
여러 사람이 같은 물건 보관함을 사용하더라도 보관함 컴퓨터는 비밀번호로 사용자를 인식하여 사용자별로 확실하게 물건을 구분해준다.
사용자A, 사용자B 모두 보관함의 컴퓨터를 통해서 물건을 보관하고, 꺼내지만 보관함의 컴퓨터가 사용자에 따라 보관 물건을 구분해준다.
```

# 2. 특징
```text
쓰레드 로컬은 사용하면 각 쓰레드마다 별도의 내부 저장소를 제공한다.
 -> 따라서 같은 인스턴스의 쓰레드 로컬 필드에 접근해도 아무 문제가 없다.
 
 Thread_A가 userA라는 값을 저장하면 쓰레드 로컬은 thread-a 전용 보관소에 데이터를 안전하게 보관한다.
 Thread_B가 userB 값을 저장하면 쓰레드 로컬은 thread-b 라는 전용 보관소에 데이터를 따로 보관하게 된다.
 
 -> 쓰레드마다 보관소가 다르기에 요청이 동시에 들어오더라도 정확하게 데이터를 반환할 수 있다.
 
 자바는 언어차원에서 쓰레드 로컬을 지원하기 위해 java.lang.ThreadLocal 클래스를 제공한다.
```
